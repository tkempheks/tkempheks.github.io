Denver, Dev and I did a spoke about Canarytokens at BSIDES Cape Town 2022. A portion of the talk covers how you could use Canarytokens to protect your SOHO networking equipment. This is what you’ll find written up below.

Detecting failed authentication attempts against a Mikrotik

Canarytokens are mostly built on top of two primitive alerting channels. HTTP and/or DNS requests. We can easily deploy various pre-built tokens (MS-Word, AWS, etc…), build our own detection and use either the Web Bug or DNS Canarytoken as a triggering mechanism.

This is what we’ll do below (Use a DNS Canarytoken to trigger alerts for failed login attempts on a Mikrotik router).

A Mikrotik script, scheduler and applicable logging needs to be configured on your Mikrotik.

Step 1 - Create Canarytoken and modify script parameters

The mikrotik_canarytoken_alerter script requires you to create a DNS Canarytoken on Canarytokens.org:

Modify the tokenDNSUrl variable to include the host name generated from the DNS Canarytoken.

Take note of the currentBuf variable which indicates that a ‘canary’ logging action is required.

Step 2 - Configure logging

A logging rule and action can be configured using the following commands:

/system logging action
add name=canary target=disk
/system logging
add action=canary topics=system,error,critical

At this point, if a log entry contains “login failure” and the script is run, an alert will be generated and sent to the email when creating your Canarytoken. In the next step, we configure a scheduler to automatically check for these entries, which subsequently triggers the alert.

Step 3 - Configure scheduler

A system scheduler can be configured using the following command:

/system scheduler
add interval=5m name=check-failed-login-attempts on-event="/system/script/run auth-alert" policy=ftp,reboot,read,write,policy,test,password,sniff,sensitive,romon start-time=startup

Step 4 - Test and confirm

Generate some failed login attempts and verify within the mikrotik log:

log/print

18:50:44 system,error,critical login failure for user root from 192.168.88.158 via ssh
18:50:44 system,error,critical login failure for user root from 192.168.88.158 via ssh
18:50:45 system,error,critical login failure for user root from 192.168.88.158 via ssh
